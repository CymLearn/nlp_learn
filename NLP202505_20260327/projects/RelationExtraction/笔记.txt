NOTE:
    实体识别中最麻烦的是嵌套实体，针对实体嵌套，BIO/BMESO序列标注模型是不太好解决该问题，可以考虑基于span的分类网络或者指针网络来实现
    关系抽取中最麻烦的是关系重叠 --> 也可以考虑span片段网络或者指针网络来实现
        Single Entity Overlap (SEO) 单一实体重叠：两个三元组之间有一个实体重叠
        Entity Pair Overlap (EPO) 实体对重叠 ：即一个实体对之间存在着多种关系
        Subject Object Overlap (SOO) 主客体重叠 ：既是主体，又是客体
关系抽取:
    定义/功能：在实体识别之后，判断实体之间存在什么类型的关系 ---> 也就是对实体组/实体对进行分类操作
    基本概念:
        实体1 --关系--> 实体2
        关系的表达，一般采用三元组来表达: (实体1,关系,实体2)
        实体1我们称为主体，关系是从这个实体出发的, eg: 姚明 --妻子--> 叶莉，姚明就是主体
        实体2我们成为客体，关系是到这个实体的，eg: 姚明 --妻子--> 叶莉，叶莉就是客体
            (姚明，妻子，叶莉)  姚明 --妻子--> 叶莉
            (叶莉，丈夫，姚明)  叶莉 --丈夫--> 姚明
            (姚明，身高226cm)   姚明 --属性--> 身高226cm
    eg:
        今年暑假，我准备去西安、新疆这两个地方去玩，但是不知道能不能从上海坐火车过去?
        假定有一个命名实体的模型，最终输出为:
            时间：今年暑假
            地名: 西安、新疆、上海
            交通方式: 火车
        关系类别列表: 无关系、出发地&目的地关系、出发地&交通方式关系、交通方式&出发时间关系
        关系抽取的最终期望结果:
            上海 --出发地&目的地关系--> 西安
            上海 --出发地&目的地关系--> 新疆
            西安 --无关系--> 新疆
            新疆 --无关系--> 上海
            上海 --出发地&交通方式关系--> 火车
            西安 --无关系--> 火车
            新疆 --无关系--> 火车
            .....
        能不能构建一个这样的模型(在实体识别的基础上构建):
            输入: 原始文本, 实体1, 实体2
            输出: 实体1和实体2之间的关系(并且要求实体1是主体)， eg: 无关系、出发地&目的地关系、出发地&交通方式关系、交通方式&出发时间关系....
            eg:
                orig_text, 上海, 西安                出发地&目的地关系
                orig_text, 上海, 新疆                出发地&目的地关系
                orig_text, 西安, 新疆                无
                orig_text, 西安, 上海                无
                orig_text, 新疆, 西安                无
                orig_text, 新疆, 上海                无
                .......
            eg:
                orig_text, 上海, 西安

spert:（片段网络）
    最核心的思路/流程：
        a. 从给定文本中进行窗口滑动，得到所有的“候选实体片段”；
            ---> 训练的时候，候选实体片段分为两类(正样本<具体实体类别>、负样本)；推理的时候暴力遍历所有候选片段，需要通过模型判断属于正样本还是负样本
        b. 将原始文本输入bert模型，得到每个token对应的特征向量 --> 模型迁移；
        c. 实体的分类判断：针对每个 "候选实体片段" 提取对应的特征向量(token片段的特征向量、片段长度特征向量、文本特征向量)，对提取的特征向量进行全连接，得到预测属于各个类别的置信度
            ---> 训练的时候，基于置信度和实际的标签值计算损失；推理的时候，基于置信度进行过滤，得到属于实体的片段(实体片段/正样本)
        d. 基于实体的正样本(实体片段)，进行交叉组合得到所有的"候选实体-关系组合"
            ---> 训练的时候，由于实体片段类型是已知的，所以可以直接构造
            ---> 推理的时候，需要基于实体预测的结果进行构造
        e. 关系的分类判断：针对每个 "候选实体-关系组" 提取对应的特征向量(实体1 bert特征向量+长度特征、实体2 bert特征向量+长度特征、实体1和实体2之间的token bert特征向量)， 对提取的特征向量进行全连接，得到预测属于各个关系类别的置信度
            ---> 训练的时候，基于置信度和实际的标签值计算损失(多分类)；推理的时候，基于置信度/概率进行过滤，得到属于关系的实体组合
    参考:
        https://www.modb.pro/db/472588
        https://github.com/lavis-nlp/spert
        https://blog.csdn.net/Wang_XB_3434/article/details/131345012
        https://arxiv.org/pdf/1909.07755.pdf
    步骤:
        -1. 从github上下载代码
            git clone https://github.com/lavis-nlp/spert.git
        -2. 导入PyCharm中，将spert文件夹添加到sys.path中
        -3. 创建一个新的分支、设定python执行环境
        -4. 下载/准备训练数据:
            参考: scripts/fetch_datasets.sh
        -5. 训练代码:
            NOTE: 需要能够访问外网
            python spert.py train --config configs/example_train.conf
            针对其它数据进行训练：（针对自己的数据集，需要将数据格式进行适配转换: duie_data_process.py）
                python spert.py train --config configs/duie_train.conf
        -6. 模型评估:
            python spert.py eval --config configs/example_eval.conf
        -7. 模型应用/推理
            python spert.py predict --config configs/example_predict.conf
    代码说明：
        configs: 内部给定了训练、评估、预测的配置文件
        data: 训练数据、日志以及训练的结果模型
        scripts: linux上的下载数据以及训练好的spert模型的脚本
        spert: 核心代码
        args.py: 给定入参定义及解析
        config_reader.py: 给定配置文件及参数的解析代码
        spert.py: 入口类
    缺陷：
		1. 非常强的依赖span片段大小，而且span的片段大小是先验值，针对异常的情况，很难覆盖
		2. 当span先验长度给定比较长的时候，模型的计算量会增加
		3. 长度会影响实体预测过程中候选实体的数量，从而影响模型的计算量,最终影响实体的预测结果；
		4. 预测为实体的数量会影响关系预测过程中，候选关系的数量，从而影响关系的计算量；
    可选的优化点:
        a. 候选实体的产生方式上
        b. 损失上的优化
        c. 网络结构上
