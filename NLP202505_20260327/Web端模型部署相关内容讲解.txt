NOTE: 一般情况下，我们可以通过对外提供一个http接口完成部署操作，这个http接口接收调用方传递过来的参数(特征数据)，内部调用模型进行预测推理，并将预测结果返回调用方
一般情况下，模型部署的流程如下:
    -1-. 训练好模型后，首先编写好样本的推理预测相关的方法，主要包括两个方法:
        模型恢复的相关方法：推理对象的构造方法，eg:__init__
        模型预测方法：接收入参，入参处理，调用模型，结果处理返回
    -2-. 在推理预测代码的外部套一层"壳"，这层"壳"相关的代码就是一些接收调用方传递特征数据、调用预测代码等相关逻辑
        eg: 本地测试的命令行输入、HTTP接口(Flask、FastAPI)、RPC接口......-->具体看公司的整体技术框架
    -3-. 相关的测试工作 --> 自己本地的一些测试
    -4-. 选择不同的部署方式，将代码 + 模型文件上传进行部署操作
        eg: 裸服务器的部署、Docker镜像部署、各种第三方的人工智能平台的部署....
        PS: 一般情况下仅需要编写一个Docker项目配置文件(Dockerfile)
        PS: 代码一般情况下使用git进行管理
    -5-. 部署好的服务进行测试check，输出一些文档即可

        https://www.runoob.com/http/http-methods.html
        https://www.runoob.com/html/html-url.html
    URL路径的解析:
        https://github.com[:port]/pallets/flask[?k1=v1&k2=v2]
            https: 协议schema，eg: http、https
            github.com: 服务器位置，域名、主机名或者ip地址
            /pallets/flask: 资源路径/路由，最终决定具体由那个方法处理当前请求
            port: 可选端口，默认http 80, https 443
            ?k1=v1&k2=v2: get请求时候的参数,此时表示有两个参数k1和k2
        HTTP的请求方式主要有Get、Post
    特殊IP地址:
        0.0.0.0: 表达当前机器的所有IP地址，但是不支持互联网访问0.0.0.0
        127.0.0.1: 当前机器的IP地址

Docker
    简单来理解的话，可以将Docker看成是一种比较方便进行部署、运维管理的软件
        image/镜像：理解成是一个包含某个服务/功能的安装包，内部包含了操作系统、服务/功能软件;
        镜像仓库：用来保存构建好的image/镜像的网站；
        container/容器：一个image镜像在docker软件中运行后最终形成的"微小的服务器" --> 其实就是一个进程
        宿主机：运行container容器的服务器/机器

    分步骤构建docker镜像并启动服务:
        NOTE:
            宿主机：Docker软件运行的机器 或者 容器运行的机器
            容器：Docker启动的镜像所形成的容器
        1. 拉取基础镜像(宿主机内执行)
            去镜像仓库进行检索:
                https://hub.docker.com/
            docker pull python:3.12
        2. 运行镜像，形成容器，并且进入容器内部(宿主机内执行)
            docker run -ti python:3.12 /bin/bash
            run命令会进入docker容器内部，在容器内部就相当在一个操作系统里面，你可以在操作系统内进行你想操作的任何内容， exit就是推出镜像
            重新启动退出关闭的容器
            docker restart <CONTAINER ID>
            # 重新进入容器内部
            docker exec -ti <CONTAINER ID> /bin/bash
        3. 在容器内部进行依赖库的安装(容器内执行)
            pip install flask==3.1.1 flask-cors==6.0.2 transformers==4.57.3 jieba onnxruntime==1.22.0 fastapi==0.109.0 uvicorn==0.40.0 -i http://mirrors.cloud.aliyuncs.com/pypi/simple/ --trusted-host mirrors.cloud.aliyuncs.com
        4. 将模型代码及模型文件从宿主机中copy到容器里面（宿主机里面执行）
            docker cp <宿主机源路径> <容器id>:<容器内部目标路径>
            docker cp text_classify ee65d5ac0660:/opt/text_classify
        5. 复制代码及模型完成后，可以在容器内部测试一下代码环境是否正常(容器内执行)
            # 启动服务
            python /opt/text_classify/test/server_runner.py
            # 使用docker exec命令重新启动一个界面，进行测试
            curl http://127.0.0.1:9991
            # 如果成功的情况下，直接退出容器；如果失败，那么针对失败内容进行解决
            exit
        6. 将容器转换为镜像(宿主机中执行)
            docker commit <容器id> <镜像名称>:<镜像版本>
            docker commit ee65d5ac0660 nlp_text_classify:v2601092109
        7. 启动镜像(宿主机执行)
            docker run -d -p 9001:9001 nlp_text_classify:v2601092109 python /opt/text_classify/test/server_runner.py
            docker run -d -p 9002:9001 nlp_text_classify:v2601092109 python /opt/text_classify/test/server_runner.py
            docker run -d -p 9003:9001 nlp_text_classify:v2601092109 python /opt/text_classify/test/server_runner.py
